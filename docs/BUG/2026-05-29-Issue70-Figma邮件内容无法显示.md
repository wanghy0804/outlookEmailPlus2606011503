## 问题描述

**来源**：[GitHub Issue #70](https://github.com/ZeroPointSix/outlookEmailPlus/issues/70)

**现象**：
- 大部分邮件都能正常显示内容
- Figma 发来的邮件（重置密码、登录验证等）**只能看到邮件头，正文空白**
- 导致无法获取验证码，无法完成密码重置等操作

**版本**：v2.5.0

---

## 根因分析

### 1. 问题定位

在 `outlook_web/services/imap.py` 第 52-83 行，`get_email_body()` 函数的逻辑有缺陷：

```python
def get_email_body(msg) -> str:
    """提取邮件正文"""
    body = ""
    if msg.is_multipart():
        for part in msg.walk():
            content_type = part.get_content_type()
            content_disposition = str(part.get("Content-Disposition", ""))

            if content_type == "text/plain" and "attachment" not in content_disposition:
                try:
                    payload = part.get_payload(decode=True)
                    charset = part.get_content_charset() or "utf-8"
                    body = payload.decode(charset, errors="replace")
                    break  # ❌ 找到 text/plain 就立即返回
                except Exception:
                    continue
            elif content_type == "text/html" and "attachment" not in content_disposition and not body:
                # ...
```

**问题**：找到 `text/plain` 就立即 `break`，即使内容为空或很短。

### 2. 为什么 Figma 邮件会出问题？

**Figma 邮件的特点**：
1. `text/plain` 部分可能几乎为空（或只有很短的内容）
2. 正文内容都在 `text/html` 部分
3. 这是很多现代邮件服务的常见做法

**当 `text/plain` 为空时**：
- `get_email_body()` 返回空字符串
- 前端显示空白

### 3. 用户 ikxd 的发现

用户 ikxd 在 Issue #70 中提供了分析：
> "我用脚本直接读取邮件，发现内容也是空的，AI解释：最大概率是邮件 plain part 被解析成空字符串（Figma 经常 plain 段几乎为空，正文都在 HTML 里），所以应该是解析的问题。"

并提供了解决方案脚本，核心逻辑：
```python
body = plain_text if len(plain_text) >= 20 else (html_text or plain_text)
```

## 修复方案

### 修改文件

**`outlook_web/services/imap.py`**

#### 1. 修改 `get_email_body()` 函数

**修改前**：
- 找到 `text/plain` 就立即返回
- 如果 `text/plain` 为空，返回空字符串

**修改后**：
- 同时提取 `text/plain` 和 `text/html`
- 如果 `text/plain` 太短（<20字符），回退到 `text/html`
- 参考用户 ikxd 的脚本逻辑

#### 2. 新增 `get_email_body_and_type()` 函数

返回 `(body, body_type)` 元组，用于需要区分 HTML/Text 的场景。

#### 3. 修改 `get_email_detail_imap_with_server()` 函数

使用 `get_email_body_and_type()` 替代 `get_email_body()`，返回 `body_type` 字段。
